# 流光下载器需求说明

日期：2026-06-15

## 产品定位

流光下载器是流光 App 的下载伴侣。流光负责发现、解析和播放在线资源；流光下载器负责接收用户复制出来的 m3u8 链接，并在用户确认后下载为本地 MP4 文件。

应用只面向用户自有或已获授权下载的资源，不支持也不提供 DRM 绕过、资源站聚合或访问控制规避能力。

## v0.1 范围

### 输入

- 打开 App 时自动读取剪贴板。
- 如果剪贴板内容是 `http://` 或 `https://` 开头，并且包含 `.m3u8`，首页显示检测结果。
- 用户可以手动粘贴或编辑 m3u8 链接。
- 第一版暂不支持自定义 `Referer`、`Cookie`、`User-Agent` 等请求头。

### 清晰度选择

- 如果链接是 master playlist，解析 `#EXT-X-STREAM-INF`。
- 默认选择最高分辨率；分辨率相同或缺失时选择最高 `BANDWIDTH`。
- 如果解析失败或信息不完整，回退到第一个可用播放流，并标记为自动选择。
- v0.1 暂不提供手动清晰度选择。

### 下载

- 解析 media playlist，下载所有分片。
- 支持队列添加任务，并按设置的最大并行任务数下载。
- 支持取消运行中和等待中的任务。
- 支持设置分片下载线程数。
- 通过前台服务显示下载进度和任务状态。
- 支持常见 `#EXT-X-KEY:METHOD=AES-128` 的 TS-HLS 加密流。
- 遇到 DRM、`SAMPLE-AES`、fMP4 或 BYTERANGE 时提示暂不支持。

### 输出

- 下载完成后必须合并为单个 `.mp4` 文件。
- 默认保存到系统 Downloads 下的 `liuguang-download` 目录。
- 应用内支持自定义下载目录，通过 Android Storage Access Framework 持久授权。

### 任务管理

- 下载页展示任务队列，并按全部、等待、下载中、已完成、失败状态筛选。
- 新建下载任务通过右下角 `+` 弹窗添加，不直接占用下载页主体空间。
- v0.1 暂不做任务记录持久化、失败重试、删除任务记录、删除本地文件和重命名下载文件。

## 非目标

- 不内置资源站。
- 不绕过 DRM、Widevine、FairPlay 或其他访问控制。
- 不在第一版支持批量导入。
- 不在第一版支持自定义请求头。

## 验收标准

- 剪贴板中存在 m3u8 链接时，打开 App 能自动识别。
- master playlist 能按最高分辨率/最高带宽自动选择默认清晰度。
- 下载普通未加密 HLS 后，系统 Downloads 中出现单个 MP4 文件。
- 下载 AES-128 TS-HLS 后，系统 Downloads 中出现解密合并后的单个 MP4 文件。
- 下载过程中切后台，通知栏持续显示进度。
- App 内可以通过系统目录选择器设置自定义下载目录。

## 后续版本

- Room 持久化任务列表和未完成任务状态。
- 暂停、继续和失败重试。
- 手动选择清晰度。
- 删除任务记录、删除本地文件和重命名下载文件。
- 自定义 `Referer`、`Cookie`、`User-Agent` 等请求头。
